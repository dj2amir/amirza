# Dockerize mirzabot

## Goal
Create a Docker-based deployment so the bot can run on any Docker host without requiring a fresh VPS. This is a **separate contribution** ‚Äî install.sh remains untouched.

## Architecture

```
docker-compose.yml
‚îú‚îÄ‚îÄ app (php:8.2-apache + supervisord for cron)
‚îÇ   ‚îú‚îÄ‚îÄ Apache serving the PHP app
‚îÇ   ‚îú‚îÄ‚îÄ Cron daemon (via supervisord)
‚îÇ   ‚îî‚îÄ‚îÄ docker-entrypoint.sh (generates config.php from env vars)
‚îÇ
‚îú‚îÄ‚îÄ db (mysql:8.0 official image)
‚îÇ   ‚îî‚îÄ‚îÄ Persistent volume for data
‚îÇ
‚îî‚îÄ‚îÄ phpmyadmin (optional, on port 8081)
    ‚îî‚îÄ‚îÄ Web-based DB management
```

**Why supervisord?** The bot admin UI dynamically adds/removes crontab entries via `shell_exec('crontab ...')`. Cron must run in the same container as PHP so they share crontab state.

**Why not a reverse proxy in compose?** SSL termination is best handled by an external reverse proxy (nginx/caddy/traefik) ‚Äî standard Docker pattern. We document this in README.

## Files to Create (6 new files)

### 1. `Dockerfile`
- **Base:** `php:8.2-apache`
- **Install:** cron, supervisord, unzip, all PHP extensions (mysql, mbstring, zip, gd, curl, intl, xml, bcmath, soap, ssh2)
- **Enable:** Apache mod_rewrite (for .htaccess)
- **Copy:** app code to `/var/www/html/`
- **Add:** supervisord.conf, docker-entrypoint.sh
- **Entrypoint:** `/usr/local/bin/docker-entrypoint.sh`

### 2. `docker-entrypoint.sh`
- Wait for MySQL to be ready (retry loop)
- Generate `/var/www/html/config.php` from environment variables
- Ensure `storage/cache/` directory exists
- Exec supervisord (runs Apache + cron)

### 3. `docker-compose.yml`
- **app:** builds from Dockerfile, maps port 8080:80, depends on db, env_file .env, volumes for storage/data/logs
- **db:** mysql:8.0, env vars from .env, volume for persistent data
- **phpmyadmin:** phpmyadmin:latest, port 8081:80, connects to db
- Named volumes: `db-data`, `app-storage`, `app-data`, `app-logs`

### 4. `.env.example`
All config values with comments:
- `DB_HOST=db`, `DB_NAME=mirzaprobot`, `DB_USER`, `DB_PASS`, `DB_ROOT_PASS`
- `TELEGRAM_TOKEN`, `ADMIN_ID`, `DOMAIN`, `BOT_USERNAME`

### 5. `supervisord.conf`
Runs `apache2-foreground` and `cron -f` as supervised processes.

### 6. `.dockerignore`
Exclude `.git/`, `.mimocode/`, `.env`, `*.log`, etc.

## Config Strategy

`config.php` currently has hardcoded placeholders:
```php
$dbhost = '{database_url}';
$dbname = '{database_name}';
$APIKEY = '{API_KEY}';
```

The entrypoint script will generate a real `config.php` by substituting these from environment variables. The original `config.php` with placeholders stays in the repo as-is (for bare-metal installs).

---

## Cron Handling

- Cron jobs are managed by the admin panel via `shell_exec('crontab ...')` commands
- `cron` daemon runs inside the app container via supervisord
- Cron scripts in `cronbot/` use `php <script>.php` ‚Äî works as-is in Docker
- No modifications needed to any cron-related code
